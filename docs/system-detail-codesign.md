# 系统详细协同设计（Detail Co-Design）

> 原文件名：`system-newdesign1.md`
> 用途：保留 V2 架构的详细设计推演、分层职责、ADR 与演进路线。

## 1. 设计哲学（详细版）

- **LLM 主导理解**：意图解析、风格映射、物理一致性检查由 LLM 直接推理
- **模型中立**：不绑定单一模型，Kimi 为默认参考实现
- **RAG 辅助**：仅用于精确参数、禁忌词、高频更新知识
- **分层解耦**：创意理解与技术约束分离
- **多模态原生**：支持文本、图像、视频帧等混合输入

## 3. 模块详细分工

### 3.1 `src/llm_plugins/`

- `base_client.py`：插件统一抽象接口
- `kimi_client.py`：默认实现
- 后续可扩展 `claude/openai/local` 客户端

### 3.2 `src/skills/`

- `seedance2/templates/`：多场景模板
- `seedance2/constraints/`：全局/领域/模型约束
- `seedance2/examples/`：目录化多模态样例与索引
- `common/`：跨模型词典与物理规则

### 3.3 `src/tech_cache/`

- 唯一 RAG 模块，负责事实性和精确性数据缓存
- 不承载创意生成逻辑

### 3.4 `private/`

- 本地私有配置、私有 Skill、私有知识
- 默认 `.gitignore` 忽略，不进入远端仓库

## 4. ADR（关键决策记录）

- ADR-001：意图解析必须用 LLM 推理，不走 RAG
- ADR-002：RAG 限定在技术参数和高频更新知识
- ADR-003：约束分层并支持本地覆盖
- ADR-004：插件化保证模型中立与可替换
- ADR-005：多模态样例目录化，替代单文件 few-shot
- ADR-006：样例索引与样例内容解耦

## 6. 实施约束（当前阶段）

- 目标是“可编译、可运行”的基础骨架
- 不接入真实 API，不实现业务功能
- 模块以 no-op 占位返回为主，优先稳定结构
